PWP Milestone 1

Purpose, Audience & Goal

Purpose: I want my PWP to enable developers to be able to create functional email forms on their static website without having to build a backend. The website is a landing page to demonstrate how to use this API endpoint.

Audience: Developers who are building static websites and need a way to collect user information. Employers who are impressed that I actually created a usable product.

Goal: The website will explain what the api does and how to use it. The website will show example code snippets, and a codepen playground that allows the user to edit and test out the api. This service will offer a straightforward solution for developers to create functional email forms on their static websites. Currently, there aren't many free options available for developers hosting static sites on platforms like GitHub Pages or Cloudflare Pages to collect contact information from users. The goal of this project is to impress potential employers and land a job. Regardless of the outcome, I want to build a tool that I can continue to improve and potentially monetize in the future. For example, I could offer a premium version with features like an email log dashboard, tables for managing collected form data, and additional email templates for paying customers. The possibilities are limitless. Depending on user adoption, I could even turn this into a profitable business, or sell the platform to another competitor.

Persona

Name: John Doe

Age: 25

Profession: Web developer student.

Technology Macbook Air, with the latest OS, and a m3 chip.

Attitudes and Behaviors: Overloaded with course material, he doesn't have the time or energy to create a backend service for his contact form. Oh... one of his classmate is developing a solution? Ok, lets call upon their api.

Frustrations and Needs: John is developing a personal portfolio. He implemented an email form, because that's what everyone else did. He implemented the form, then he gets to the submit button, but it doesn't work. He can't figure out how to send himself an email. He asks the instructor. The instructor is like "bro you need a back end server to handle the submission". Lol, what?

Goals: John wants to submit forms without the headache of creating a backend server. So he found this easy to use, one line api integration that literally just works.

User Story: As a user, I want to get my forms working with just 1 line of code.
